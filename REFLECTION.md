# Cookie Clicker Reflection

## What helped you understand DOM interaction best?

Using `document.getElementById()` helped me understand DOM interaction the most. I used it to select the button, counter, and message elements from the HTML. Then JavaScript was able to update the page using `textContent`. Seeing the counter change every time I clicked the button made DOM interaction easier to understand.

## How did you choose your milestone messages?

I chose milestone messages that become more humorous as the number of clicks increases. The messages start with encouragement at 10 clicks, become more surprised at 25 and 50 clicks, and end with a reminder to take a break at 100 clicks.

## What challenge or bug surprised you?

One challenge was making sure the event listener was connected to the correct button and that the counter increased with every click. I also had to make sure the milestone conditions used the correct click numbers so each message appeared at the right time.

## What personal twist did you add?

I added a background color change at each milestone. This gives the user visual feedback in addition to the changing message. I also used cookie and food-related emojis to make the clicker feel more fun and personalized.

## What real-world app uses this kind of interaction?

Many real-world apps use this type of event-driven interaction. Social media apps use clicks for likes, reactions, and buttons. Online shopping websites also use clicks to add items to a cart or update quantities. These apps respond to user actions using event listeners and update the page dynamically.
